PENGENALAN
JURUS (Jurutera Siber rawSEC) ialah program laluan kompetensi keselamatan siber
kebangsaan di Malaysia.
Program ini dibangunkan oleh komuniti rawSEC dengan kerjasama strategik
Cybersecurity Research Centre (CYRES), Universiti Sains Malaysia (USM).
Ia berfungsi sebagai projek rintis (pilot project) nasional untuk melatih dan menguji
generasi baharu pakar keselamatan siber tempatan.
Latihan merangkumi aspek pertahanan sistem asas sehinggalah ke tahap seni
bina ketahanan infrastruktur.

KAJIAN KES
Municipal Citizen Service Portal Project
Digital Transformation: MP Cyberjaya is launching a one-stop Citizen Service
Portal for resident complaints, requests, tracking, and announcements.
The Audit Gap: An external audit revealed that the council lacks the secure,
operational infrastructure required to back this service.
Your Role: Acting as a Cyber Engineering Consultancy (Lead Analyst) to
design and deliver a production-ready environment.
No infrastructure, tools, or templates are provided. Participants must
independently determine technologies using sound engineering judgment.


=== Page 5 ===
KAJIAN KES (SAMB.)
Timeline & Engineering Execution
Challenge Duration: Three (3) weeks total to design, build, and deploy.
Deployment Flexibility: Complete freedom to use local virtualization (VMs), cloud
environments, or bare-metal physical hardware.
Participants are fully responsible for:
• Infrastructure provisioning and software installation.
• Security configuration, baseline baseline auditing, and system testing.
• Hands-on troubleshooting, logging verification, and live demo preparation.

=== Page 8 ===
CADANGAN SOLUSI
Kejuruteraan Sistem & Rangkaian (OS & Network Hardening)
Sistem Operasi: Linux Ubuntu Server 24.04 LTS atau Linux Mint (Virtual Machine
dalam VMware/Proxmox).
Kawalan Akses & SSH:
• Matikan login root secara terus (PermitRootLogin no).
• Wajibkan pengesahan berasaskan kunci awam (Key-based Authentication)
sahaja. Tukar port default SSH (contoh: port 2222).
Host Firewall (ufw): Set polisi default untuk menyekat semua trafik masuk (sudo ufw
default deny incoming). Hanya buka port web (HTTP/HTTPS) dan port SSH tersuai
secara bersasar.


=== Page 9 ===
CADANGAN SOLUSI (SAMB.)
Keselamatan Aplikasi (Application Security )
Web Server / Reverse Proxy: Nginx.
Hardening Lapisan Web:
• Sembunyikan banner versi perisian (server_tokens off; dalam Nginx).
• Konfigurasi SSL/TLS menggunakan sijil self-signed atau Let's Encrypt (dengan TLS
1.3 sahaja, matikan protokol usang seperti TLS 1.0/1.1).
• Masukkan pengepala keselamatan (Security Headers) seperti X-Frame-Options,
Content-Security-Policy, dan X-Content-Type-Options.


=== Page 10 ===
CADANGAN SOLUSI (SAMB.)
Keselamatan Pangkalan Data (Database Security )
Teknologi: PostgreSQL atau MariaDB.
Hardening Pangkalan Data:
• Least Privilege: Cipta pengguna (user) khusus untuk portal web dengan hak
akses terhad (hanya SELECT, INSERT, UPDATE pada jadual tertentu). Jangan
sekali-kali gunakan akaun root atau postgres dalam kod aplikasi.
• Host Access Management: Konfigurasi fail pg_hba.conf atau my.cnf supaya
pangkalan data hanya menerima sambungan daripada hos tempatan (localhost /
standard Unix socket), bukan daripada IP luar.


=== Page 11 ===
CADANGAN SOLUSI (SAMB.)
Pemantauan Keselamatan (Security Monitoring / Observability)
Fail2ban: Dipasang untuk memantau log SSH dan Nginx. Jika terdapat cubaan log
masuk gagal melebihi 5 kali, IP tersebut akan disekat (banned) secara automatik
melalui iptables.
Log Management: Gunakan Rsyslog atau Logwatch untuk mengumpulkan log sistem
yang kritikal, diletakkan di bawah pengawasan ketat dengan konfigurasi log rotation
yang selamat.


=== Page 12 ===
CADANGAN SOLUSI (SAMB.)
Business Continuity & Recovery (BCP/DR)
Automated Backup: Sediakan skrip Bash (backup.sh) yang dijalankan secara
automatik menggunakan Cron Job.
Mekanisme: Skrip akan melakukan database dump (pg_dump atau mysqldump),
memampatkan fail fail aduan warga, dan menyimpannya ke direktori sandaran
tersendiri lengkap dengan tanda masa (timestamp).
Pengukuran Parameter: Simulasikan pemulihan untuk mengira masa bertindak
secara nyata bagi mematuhi had RTO (Recovery Time Objective) dan RPO (Recovery
Point Objective).


=== Page 13 ===
CADANGAN MAKLUMAT DALAM LAPORAN
1. Executive Summary & Architecture: Gambar rajah topologi rangkaian logikal, huraian
persekitaran (VM/Cloud), spesifikasi perkakasan dan justifikasi pemilihan OS - Diagram
dan Teks.
2. Operating System & Network: Bukti konfigurasi pelayan, polisi sudoers, pengerasan SSH
Daemon, dan rantaian peraturan Firewall (UFW/Iptables)– Fail konfigurasi dan kod bash
3. App & Database Lockdown: Kebenaran fail konfigurasi pangkalan data, bukti sekatan akses
port DB, tetapan SSL/TLS Nginx, dan laporan ujian tajuk HTTP – Screenshot dan Kod
Konfigurasi
4. Logging & Alerting System: Konfigurasi pemantau log (Syslog/Wazuh/Fail2ban) dan log
amaran aktif yang memaparkan cubaan serangan dikesan secara langsung – Ekstrak Fail
Log.
5. BCP/DR Strategy & Scripts: Kod skrip sandaran automatik, kunci enkripsi, dan pembuktian
langkah pemulihan (restore) pangkalan data berpandukan parameter RTO/RPO – Skrip
bash dan jadual RTO


=== Page 14 ===
30 MINIT PEMBENTANGAN
1. Minit 0–7 (Seni Bina & Rationale): Bentangkan rajah infrastruktur anda. Nyatakan
mengapa teknologi tersebut dipilih dan apa trade-off yang anda ambil (Contoh: memilih
persekitaran tunggal atas faktor kawalan kos dan kekangan masa 3 minggu).
2. Minit 8–20 (Live Pressure Test / Walkthrough): Tunjukkan bukti sistem berfungsi secara
langsung di hadapan penilai: Lakukan vulnerability scanning ringkas untuk
membuktikan banner versi pelayan berjaya disembunyikan. Tunjukkan log iptables/ufw
yang menapis trafik masuk. Pamerkan akaun pangkalan data yang tidak boleh mengakses
jadual sistem lain.
3. Minit 21–25 (Simulasi BCP/DR): Jalankan skrip backup anda secara langsung,
padamkan pangkalan data secara sengaja (simulated crash), dan lakukan proses restore
(pemulihan) untuk membuktikan sistem kembali normal dalam tempoh RTO yang
ditetapkan.
4. Minit 26–30 (Sesi Soal Jawab / Technical Defense): Bersedia mempertahankan reka
bentuk anda daripada soalan kritikal penilai mengenai pengurusan risiko siber portal
tersebut.
